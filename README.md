
# Expense Tracker (Console Application)

## Опис проєкту
**Expense Tracker** — це консольний застосунок для обліку особистих витрат з
використанням реляційної бази даних PostgreSQL.

Проєкт дозволяє:
- додавати, редагувати та видаляти витрати
- працювати з категоріями витрат (CRUD)
- переглядати аналітичні звіти
- експортувати витрати у CSV-файл за обраний період

---

## Функціональні можливості

### Категорії
- додавання категорій
- перегляд списку категорій
- редагування категорій
- видалення категорій

### Витрати
- додавання витрати (назва, дата, категорія, сума, валюта, опис)
- перегляд списку витрат
- перегляд детальної інформації по витраті
- редагування витрати
- видалення витрати

### Звіти
- витрати за період
- фільтрація витрат за **назвою витрати (expenses.title)**
- витрати по конкретній категорії (**WHERE category_id = %s**)
- мінімальні та максимальні витрати
- підсумки по категоріях
- експорт витрат за період у CSV

---

## Обрана СУБД
У проєкті використовується **PostgreSQL**.

Причини вибору:
- підтримка реляційної моделі даних
- забезпечення цілісності даних
- зручна робота з Python через `psycopg2`
- підтримка зовнішніх ключів та складних SQL-запитів

## Налаштування середовища

### 1. Клонування репозиторію
```bash
git clone https://github.com/olgagreb/expense-tracker-exam.git
cd expense-tracker-exam
```

### 2. Створення віртуального середовища
```bash
python -m venv .venv
.venv\Scripts\activate
```

### 3. Встановлення залежностей
```bash
pip install -r requirements.txt
```

### 4. Створення БД PostgreSQL
```sql
CREATE DATABASE expense_exam;
```

### 5. Налаштування `.env`
Створіть файл `.env` на основі `.env.example` та заповніть свої дані:

```
DB_HOST=localhost
DB_PORT=5432
DB_NAME=expense_exam
DB_USER=postgres
DB_PASSWORD=your_password
```
