
# Expense Tracker (Console Application)

## Опис проєкту

**Expense Tracker** — це консольний застосунок для обліку особистих витрат з використанням реляційної бази даних **PostgreSQL**.

Проєкт виконано як екзаменаційну роботу та він демонструє:
- роботу з реляційною БД
- реалізацію CRUD-операцій
- аналітичні SQL-звіти
- структурований та відрефакторений Python-код
- роботу з конфігураціями та логуванням

---

## Функціональні можливості

### Категорії витрат
- створення категорій
- перегляд категорій
- редагування категорій
- видалення категорій  
(CRUD)

### Витрати
- додавання витрат (назва, дата, сума, валюта, категорія, опис)
- перегляд усіх витрат
- редагування витрат
- видалення витрат  
(CRUD)

### Звіти та аналітика
- витрати за вибраний період
- фільтрація витрат за **назвою витрати**
- витрати по конкретній категорії (`category_id`)
- максимальна витрата у кожній категорії
- мінімальна витрата у кожній категорії
- максимальна витрата за вибраний період
- мінімальна витрата за вибраний період
- підсумки по категоріях за період (сума)
- топ-категорія за період (де витрат найбільше)
- середні витрати на день за період

### Додатково
- експорт витрат у CSV-файл
- логування дій застосунку у файл
- зберігання параметрів БД у `.env`

---

## Обрана СУБД

У проєкті використовується **PostgreSQL**, оскільки:
- підтримує реляційну модель даних
- дозволяє використовувати `FOREIGN KEY`
- підтримує агрегатні функції (`SUM`, `MIN`, `MAX`, `AVG`)
- добре працює з Python через `psycopg2`

## Налаштування середовища

### 1. Клонування репозиторію
git clone https://github.com/olgagreb/expense-tracker-exam.git  
cd expense-tracker-exam

### 2. Створення та активація віртуального середовища
python -m venv .venv  
.venv\Scripts\activate

### 3. Встановлення залежностей
pip install -r requirements.txt

---

## Налаштування бази даних

1. Створити базу даних PostgreSQL:
CREATE DATABASE expense_exam;

2. Створити файл `.env` на основі `.env.example`:
DB_HOST=localhost  
DB_PORT=5432  
DB_NAME=expense_exam  
DB_USER=postgres  
DB_PASSWORD=your_password  

Файл `.env` не зберігається у репозиторії з міркувань безпеки.

## Логування

У проєкті реалізовано логування:
- всі ключові дії записуються у файл `logs/app.log`
- використовується стандартний модуль `logging`
- логування дозволяє відслідковувати роботу програми та можливі помилки
